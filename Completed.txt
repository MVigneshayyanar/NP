Sensory AI — Full-Stack Implementation Plan
Build a Go Fiber gateway + Django backend + Flutter mobile app for sensory room analysis, all orchestrated with Docker Compose.

User Review Required
IMPORTANT

Platform Choice — Flutter recommended. Your spec mentions both Jetpack Compose and Flutter. Given that your existing codebase (max-my-bill-1) is Flutter/Dart, and Flutter gives you iOS + Android from one codebase, I'll proceed with Flutter. Confirm or redirect.

IMPORTANT

LLM Provider. The spec references a "Multimodal LLM API" but doesn't name a provider. I'll stub the pipeline with an OpenAI-compatible API (GPT-4o / GPT-4 Vision style). The Go gateway will call it via REST. You can swap to Gemini, Claude, etc. later by changing the adapter. Please confirm or specify a provider + API key handling preference.

IMPORTANT

Paint Database API. Section 2.4 requires a 3rd-party paint database for color matching. There is no widely available free paint API. I'll build a local seed database of ~200 popular paint colors (Benjamin Moore / Sherwin-Williams style names + hex + LRV) and a nearest-color matching algorithm. You can plug in a real API later. Confirm this approach.

WARNING

Stripe Connect. Section 2.5 requires Stripe Connect for royalty payouts. I'll build the models, anonymization pipeline, and API stubs but will NOT connect to live Stripe in this initial build (requires your Stripe platform account, onboarding flow, and legal review). The code will be wired to a STRIPE_CONNECT_ENABLED=false feature flag.

Open Questions
JWT Secret Sharing: Django issues JWTs, Go verifies them. I'll use a shared JWT_SECRET env var in Docker Compose. Is RS256 (asymmetric) preferred, or is HS256 (symmetric shared secret) acceptable for now?
→ Default: HS256 for simplicity, can migrate to RS256 later.

Audio Processing: The spec says "ambient audio or audio-derived features." Should Go extract audio features (dB level, frequency spectrum) before sending to the LLM, or send raw audio? → Default: Extract basic dB level + duration in Go, send metadata to LLM (most multimodal LLMs don't accept raw audio well). Raw audio stored for future processing.

Deployment Target: Docker Compose for local dev is clear. Any cloud target in mind (AWS ECS, GCP Cloud Run, etc.) for production?
→ Default: Build for Docker Compose only; production deployment is out of scope for this phase.
